# Skill corpus audit — 2026-07-07

Audit of every skill installed on this machine, run to ground ai-shakespii's lint rules and harness design in real defects rather than theory.

**Scope:** 13 personal skills (`~/.claude/skills/`), 14 superpowers 6.1.1 skills (active installed version), skill-creator, plugin-dev `skill-development`, plus reference files (`anthropic-best-practices.md`, `testing-skills-with-subagents.md`, `persuasion-principles.md`, skill-creator `schemas.md`).

**Headline:** zero versioning anywhere (1/30 skills has a `version` field), zero working test harnesses (the one attempt ships broken), ~70 lines copy-pasted verbatim across 5 skills, output contracts delegated to documents that aren't shipped, and undeclared environment dependencies throughout. Even the reference corpus violates its own rules.

### S1–S6: systemic weaknesses (these become lint rules)

1. **S1 — Zero versioning (13/13).** No `version`, `author`, or change tracking anywhere.
2. **S2 — No tests (12/13); the exception is broken.** Only compress attempts an eval harness and its fixtures directory is missing as installed — the benchmark numbers in its README are unreproducible.
3. **S3 — Large-scale verbatim duplication.** The ~70-line "Verify collab readiness" block (JSON shape + every error branch) is copy-pasted across bugfix / deliberation / quick-task / ralph / sdd. "Fire-and-forget", "Duo roleplay", "Resume/cancel", and "Pausing" sections are near-identical across the same five.
4. **S4 — Weak worked examples.** Most skills substitute trigger-phrase lists for real input→output pairs; only caveman, compress, find-skills have genuine before/after examples.
5. **S5 — Output contracts delegated out-of-band.** code-review, deliberation-craft, and (partially) plan-execution defer the authoritative format to an external protocol/handoff not shipped in the skill directory — the skill is not self-contained.
6. **S6 — Undeclared environment preconditions.** Hidden hard dependencies never enumerated up front: `whisper` CLI on PATH, `~/.ai-whisper/auth.json`, the `claude` binary (compress), `npx skills` + network (find-skills), `AI14ALL_*` / `CLAUDE_CODE_SESSION_ID` env vars, repo-layout output dirs (`docs/superpowers/deliberations/`, `.ai-whisper/...`).

### Overlap

- **Five near-identical kickoff skills** (bugfix, deliberation, quick-task, ralph, sdd) share ~80% of their bodies; they differ in `--type`, validation strictness, and static prose. Strong candidate to collapse into one parameterized skill + per-workflow deltas.
- **caveman ⊃ compress:** compress's Remove/Preserve rules re-derive caveman's grammar; compress is "caveman applied to files".
- **ai-whisper-code-review** overlaps the reviewer role in plan-execution's subagent loop and the global `code-review` skill.

### Standouts (best-engineered)

1. **ai-whisper-plan-execution** — explicit gating preconditions, literal handback output contract, dedicated failure-handling section, model-alias discipline (never versioned model names). Densest signal, least boilerplate.
2. **compress** — the most complete "software component" (declared I/O, backup contract, local validator, real examples, only eval attempt) despite the broken fixtures.

## Part 2 — Reference corpus (superpowers 6.1.1, skill-creator, plugin-dev)

### Structure conventions (what "professional" looks like)

- Frontmatter: universally `name` + `description` only. Sole `version:` field in the entire ecosystem: plugin-dev `skill-development` (0.1.0). Field limits: name ≤64 chars (kebab), description ≤1024 chars, frontmatter total ≤1024. Spec pointer: agentskills.io/specification.
- Descriptions: third person; 13/14 superpowers descriptions literally begin "Use when". Measured lengths 79–234 chars.
- Recurring body furniture: `## Overview` with a bolded `Core principle:`; `## When to Use` (with "When NOT"); `## Quick Reference` table; `## Common Mistakes` / `## Red Flags`; rationalization tables with exact header `| Excuse | Reality |`; `## The Iron Law` fenced ALL-CAPS rule; "Announce at start" lines; cross-skill deps as `**REQUIRED SUB-SKILL:**` bare names (never `@`-links — those force-load and burn context).
- Directory conventions: `scripts/` (executed, not loaded), `references/` (loaded on demand, one level deep, TOC required >100 lines), `assets/`.

### Quality criteria the ecosystem states (checkable)

Third-person trigger-first descriptions with searchable keywords; verb-first/gerund names, no vague names (Helper/Utils); "agents are already smart" token-cost justification for every paragraph; one excellent code example over multi-language dilution; consistent terminology; no time-sensitive info outside an "Old patterns" details block; flowcharts only for non-obvious decision points; freedom calibration (prose for open-ended, exact script for fragile); discipline skills need loophole-closing + rationalization table + red flags; scripts must solve rather than punt, document magic numbers, declare required packages.

### Gaps even in the reference corpus (ai-shakespii's exact opening)

- **No declared inputs or output contracts anywhere** — the closest things are prose conventions (subagent-driven-development's status enum, finishing-a-development-branch's exact menu text), never declared or validated.
- **No versioning, no changelogs** despite writing-skills asserting its Iron Law "applies to NEW skills AND EDITS".
- **No shipped evals.** writing-skills preaches RED-GREEN-REFACTOR for skills and ships zero fixtures; skill-creator defines the full eval schema suite and its own `evals/` directory does not exist.
- **Size rules violated by the flagships:** writing-skills is 689 lines / 3,807 words; subagent-driven-development is 419 lines / 3,085 words — both breach the ecosystem's own "<500 lines" guidance.
- **Contradicting authorities** on description style and size budgets (see STRATEGY.md D4 for shakespii's adjudication).

### Testing landscape (what exists to reuse)

- superpowers `testing-skills-with-subagents.md`: full methodology (pressure scenarios with 3+ combined pressures, no-guidance control, 5+ reps, variance-as-metric) — no fixtures.
- Anthropic best-practices: evaluation-driven development, minimum three evals, baseline-without-skill, eval JSON shape — no runner.
- **skill-creator: the most complete harness** — executable scripts (`run_eval.py`, `run_loop.py`, `aggregate_benchmark.py`, `improve_description.py`, `quick_validate.py`, `package_skill.py`, grader/comparator/analyzer agents) + full JSON schemas in `references/schemas.md`, including the trigger-accuracy eval design (20 labeled queries incl. near-miss negatives, 60/40 train/held-out split, 3 reps, select by test score).
- Net: methodology and harness both exist; **no skill commits its own test cases**. That is the gap the shakespii harness fills.
